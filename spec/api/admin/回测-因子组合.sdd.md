# SDD · 回测因子组合 CRUD

> **HTTP：**  
> - `GET /api/admin/quant/factor-combo`  
> - `POST /api/admin/quant/factor-combo`  
> - `PUT /api/admin/quant/factor-combo/{id}`  
> - `DELETE /api/admin/quant/factor-combo/{id}`  
> **模式：** ① Service 读/写（同步 JSON；配方轻量，不走 SSE）  
> **源码：** `src/api/routers/admin/quant.py`；`src/service/quant/factor_combo_service.py`

---

## 1. 概述

保存多因子「配方」供回测选用。只存元数据，**不**物化合成因子 Parquet。

---

## 2. 表 `factor_combo`

| 列 | 类型 | 说明 |
|----|------|------|
| id | serial PK | |
| name | varchar unique | 全局唯一展示名，如 `动量波动等权` |
| items | jsonb | `[{ "factor_name": "gtja_alpha1", "weight": 1.0 }, ...]` |
| remark | text nullable | |
| created_at / updated_at | timestamp | |

约束：`items` 至少 2 个元素；`factor_name` 非空；`weight` 为有限正数（允许 ≠1，回测时按权重线性组合 z-score）。

---

## 3. API

### GET 列表

返回全部组合（数量少，不分页）。

### POST 创建

Body：`{ "name", "items": [...], "remark?" }`  
名称冲突 → 400。

### PUT 更新

可改 name / items / remark；名称冲突 → 400。

### DELETE

硬删除配方；已有 `backtest_run` 保留 `combo_name` 冗余字段，不级联删历史。

---
