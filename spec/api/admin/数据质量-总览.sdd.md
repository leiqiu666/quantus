# SDD · 量化数据源 · 数据总览

> **模式：** ① Service 读（同步 JSON）  
> **HTTP：** `GET /api/admin/data-source/overview?window=5`  
> **源码：** [`src/api/routers/admin/data_source.py`](../../src/api/routers/admin/data_source.py)  
> **Service：** [`src/service/etl/completeness_overview_service.py`](../../src/service/etl/completeness_overview_service.py)

---

## 1. 概述

在现有 6 张明细宽表之上，提供**首屏总览**：跨分组聚合最近窗口完整性、待处理缺口清单、关键路径滞后与调度摘要。不替代 `POST .../dashboard` 宽表。

Admin 菜单：**量化数据源 → 数据总览**（`/data-source/overview`）。

### 触发示例

```bash
curl 'http://localhost:8000/api/admin/data-source/overview?window=5'
```

---

## 2. 调用链

| 层级 | 组件 |
|------|------|
| Router | `get_overview` |
| Service | `CompletenessOverviewService.get_overview` |
| 复用 | `CompletenessDashboardService.get_dashboard`（每分组 window 条） |
| Config | `DASHBOARD_GROUPS` + `GROUP_DETAIL_PATHS` |
| 调度 | `ScheduleRunService.recent_runs` / `ScheduleJobModel.count_enabled_jobs` |
| 关键路径 | `KlineDailyModel.get_max_trade_date` + `TradeCalService` |

---

## 3. 请求

| 参数 | 类型 | 默认 | 说明 |
|------|------|------|------|
| `window` | int | 5 | 每分组取最近 N 条 date_key，1–30 |

---

## 4. 响应字段

| 字段 | 说明 |
|------|------|
| `as_of` | 服务端今日 YYYYMMDD |
| `latest_trade_date` | 最近 SSE 开市日 |
| `is_trading_day` | 今日是否开市 |
| `window` | 实际窗口 |
| `source_total` | 6 组可监控列合计 |
| `group_total` | 分组数（6） |
| `groups_healthy` | 窗口内全绿分组数 |
| `gap_cell_count` | 窗口内未达标单元格总数 |
| `active_stock` | 基准日 listed/trading 股数 |
| `groups[]` | 分组卡片：complete_rate、status、worst_column、detail_path |
| `gaps[]` | 缺口 Top 20（含 sse_task_key） |
| `key_paths[]` | 关键路径滞后（Phase 3：日 K） |
| `scheduler` | 启用任务数、今日执行、recent_runs |

### status 枚举

- 分组：`healthy` / `warning` / `critical`
- 关键路径：`ok` / `warning` / `critical` / `unknown`

---

## 5. Admin 页面（三阶段）

| 阶段 | 能力 |
|------|------|
| Phase 1 | 全局 Statistic、6 分组健康卡片、点击跳转明细页 |
| Phase 2 | 窗口 Segmented（5/10/20）、缺口表 SSE 补位、跳转明细 `?focus=date_key` |
| Phase 3 | 关键路径滞后面板、调度今日执行摘要 + 最近 run 表 |

明细页 [`DataSourceDashboardTable`](../../../src/web/admin/src/components/DataSourceDashboard/DataSourceDashboardTable.tsx) 读取 `focus` 查询参数高亮对应行。

---

## 6. 与明细看板关系

| 接口 | 粒度 |
|------|------|
| `GET /overview` | 6 组 × 窗口聚合 + 缺口 Top N |
| `POST /dashboard` | 单组 × 分页宽表 |

配置单一事实源：[`completeness_dashboard_config.py`](../../src/service/etl/completeness_dashboard_config.py)。
