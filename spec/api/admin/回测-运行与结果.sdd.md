# SDD · 回测运行与结果

> **HTTP：**  
> - `POST /api/admin/etl/sse/run`（`task_key=backtest_run`）→ **SSE**  
> - `GET /api/admin/quant/backtest/runs` → JSON 列表  
> - `GET /api/admin/quant/backtest/runs/{run_id}` → JSON 详情  
> - `GET /api/admin/quant/backtest/runs/{run_id}/tables` → 明细表（见 [`回测-明细表.sdd.md`](./回测-明细表.sdd.md)）  
> **模式：** ② SSE 写 + ① Service 读

---

## 1. 落盘

```text
{WAREHOUSE_ROOT}/backtest/{strategy_name}/{run_id}/
  report.json / returns.parquet / ic.parquet / portfolio.parquet / trades.parquet
  turnover.parquet   # 可选：trade_date, turnover
```

---

## 2. SSE `backtest_run` 扩展字段

| 字段 | 说明 |
|------|------|
| backtest_mode / factor_name / combo_id / groups / rebalance | 同前 |
| commission_rate | 默认 0.0003 |
| stamp_duty_rate | 默认 0.001 |
| slippage_rate | 默认 0 |

基准恒为 `000300.SH`（服务端写死）。

---

## 3. 详情响应增强

在原有 `summary` / `returns` / `ic` / `nav_curves` 上增加：

| 字段 | 说明 |
|------|------|
| nav_curves.benchmark | 基准累计净值 |
| nav_curves.excess | 多空净值 / 基准净值（相对） |
| group_totals | `[{group_id, total_return, final_nav}]` |
| turnover_series | `[{trade_date, turnover}]` |
| yearly | `[{year, annual_return, sharpe, mdd, n_days}]`（long_short） |
| warnings | `string[]` |
| cost | `{commission_rate, stamp_duty_rate, slippage_rate}` |
| benchmark | `000300.SH` |

### warnings 规则

- 调仓次数 &lt; 3  
- 开市日数 &lt; 60  
- 任一日全市场 `close_adj` 收益中位数绝对值 &gt; 0.20  
- 区间内 `adj_factor` 为 NULL 的占比 &gt; 5%（若列存在）
