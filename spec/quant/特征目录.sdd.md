# SDD · 特征目录（feature_meta）

> **存储：** PG `feature_meta`（目录元数据，不物化特征值）  
> **Admin：** `/quant/feature-list`（因子管理 → 特征管理，子菜单第一位）  
> **API：** `GET/PUT /api/admin/quant/feature/*`（见 [`spec/api/admin/特征-特征列表.sdd.md`](../api/admin/特征-特征列表.sdd.md)）

---

## 1. 概述

为可编辑因子公式提供**符号表 / 血缘 / 覆盖区间**。权威行情仍在 `kline_daily` 等源 Parquet；本表只登记「公式里能用的名字」以及它们从哪来、覆盖到哪天。

国泰 191 panel 当前写死在 `gtja/engine._eval_alpha`；本目录先把这批符号入库，供 Admin 展示与后续公式校验。

---

## 2. 表结构 `feature_meta`

| 列 | 类型 | 说明 |
|----|------|------|
| id | int PK | |
| feature_name | varchar(64) unique | 公式符号，如 `CLOSE` / `RET` |
| display_name | varchar(200) | 中文名 |
| feature_kind | varchar(32) | `source` 源映射 / `derived` 派生 |
| source_kind | varchar(64) | `kline_daily` / `index_daily` / `derived` |
| source_path | varchar(200) | 物理提示：`kline_daily` / `index_daily` |
| source_column | varchar(64) | 源列，如 `close` / `vol` |
| transform | text | 变换说明，如 `close * fill_null(adj_factor,1)` |
| frequency | varchar(16) | 默认 `daily` |
| domain | varchar(32) | `stock` / `index` |
| dtype | varchar(32) | 默认 `float64` |
| formula | text | 派生特征公式（可选） |
| start_date / end_date | varchar(8) | 覆盖区间，扫描源数据写入 |
| enabled | int | 1 启用 / 0 禁用 |
| sort_order | int | 列表排序 |
| remark | text | 备注 |

冲突键：`feature_name` 唯一。

---

## 3. 种子特征（国泰 panel）

| feature_name | kind | source | 说明 |
|--------------|------|--------|------|
| OPEN/HIGH/LOW/CLOSE | source | kline → `*_adj` | 后复权 OHLC |
| VOLUME / VOL / AMOUNT | source | kline | 量、额 |
| VWAP / RET | derived | kline 派生 | panel 内计算 |
| DTM/DBM/HD/LD/TR | derived | kline 派生 | 国泰简化定义 |
| BANCHMARKINDEXOPEN/CLOSE | source | index_daily | 基准指数 |

---

## 4. 覆盖刷新

`POST /api/admin/quant/feature/refresh-coverage`：

- `source_kind=kline_daily`（及派生）：扫 `WAREHOUSE_ROOT/kline_daily` 分区，取 `trade_date` min/max
- `source_kind=index_daily`：扫 `index_daily` 或读 PG `index_daily`

不写特征值。
