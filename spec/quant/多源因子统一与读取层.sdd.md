## 1. 概述

把「自研 BaseFactor」「Tushare Research 长表」「ETL PG 旁路」三条路径定界，并补齐统一读取层 `FactorDataset`，使后续截面回测与国泰 191 只依赖一套权威存储。

| 路径 | 角色 | 本 Phase 动作 |
|------|------|----------------|
| Parquet `factor/{name}/` | **研究权威冷层** | 读取 API；不改写入格式 |
| PG `factor_meta` | 目录 | 补约定字段语义 |
| PG `factor_latest` | 近 60 日热层 | **扩展同步范围含 tushare** |
| PG `kline_stock_factor` | 运营旁路 | 文档定界，不扩展、不同步 |

## 3. Parquet Schema（不变）

```
{WAREHOUSE_ROOT}/factor/{factor_name}/dt=YYYYMM/*.parquet
```

| 列 | 类型 | 说明 |
|----|------|------|
| ts_code | Utf8 / Dictionary | |
| trade_date | Utf8 YYYYMMDD | |
| value | Float64 | |

排序建议：`(trade_date, ts_code)`。整月分区幂等 overwrite（已有 Load 行为保持）。

---

## 4. 命名冲突规则

1. `factor_name` 全局唯一（目录名 = `factor_meta.factor_name`）。
2. `FactorMetaService.update_meta` 扫描时：若同名已存在且 `source=自研`，则 **跳过** 同名 tushare 条目（或写入时不覆盖自研行）。
3. 若未来必须并存同语义不同源：Tushare 侧使用显式前缀（如 `ts_{name}`），本 Phase 默认采用「自研优先 + 跳过」即可。
4. 国泰 191 预留目录名：`gtja_alpha{N}`（N 为 1–191，无前导零或统一三位，实现时二选一写死）。

## 5. FactorDataset API

文件：`src/research/dataset/factor.py`

```python
class FactorDataset:
    def __init__(self, warehouse_root: str | None = None) -> None: ...

    def list_factors(self) -> list[str]:
        """扫描 factor/ 下一层目录名。"""

    def list_available_months(self, factor_name: str) -> list[str]:
        """返回已有 dt=YYYYMM 列表。"""

    def read(
        self,
        factor_name: str,
        start: str,
        end: str,
        ts_codes: list[str] | None = None,
    ) -> pl.LazyFrame:
        """
        读 [start, end] 闭区间（YYYYMMDD）。
        输出列：ts_code, trade_date, value。
        按月份裁剪分区后再 filter 日期。
        """

    def read_multi(
        self,
        factor_names: list[str],
        trade_date: str,
    ) -> pl.DataFrame:
        """
        单日多因子截面。
        输出：ts_code + 各因子名为列（值为 value），outer/inner join 策略：
        默认 inner on ts_code（回测可配置）；首版用 left join 以第一个因子为基亦可，
        实现时选一种并在此文档改成「已定」。
        """
```

**首版定稿：** `read_multi` 以 `factor_names[0]` 的截面为基表，其余因子 **left join** on `ts_code`，缺失为 null。

实现：`pl.scan_parquet`；不强制 DuckDB。跨源校验脚本可另用 DuckDB，非本 API 职责。

open_adj  = open  * adj_factor.fill_null(1.0)

## 7. 热层同步扩展

文件：[`src/research/factor/sync.py`](../../src/research/factor/sync.py)

### 现状

`ensure_factor_columns` / `sync_to_pg` 只遍历 `FactorRegistry.list_all()`（自研）。

### 目标

1. 列集合 = 自研注册表 ∪（`factor_meta` 中 `source=tushare` 且 Parquet 目录存在的 name）  
   或：扫描 `factor/` 目录 ∩ `factor_meta`（推荐后者，避免未入库孤儿列）。
2. 仍只保留最近 `days`（默认 60）个交易日。
3. 宽表 upsert 冲突键：`(ts_code, trade_date)`。
4. 列过多时允许 CLI 增加 `--source self|tushare|all`（可选；首版 `all`）。

验收：`factor sync-pg` 后 `factor_latest` 同时出现 `momentum_20d` 与至少一个 tushare 因子列（如 `macd`）。

## 8. factor_meta 约定补齐

不强制改表结构；在 `formula` / 现有字段中约定：

| 语义 | 落点 |
|------|------|
| 复权约定 | 文档约定全库研究因子 `adj_convention=hfq`；可写入 `formula` JSON 或 description |
| category | 自研：`price_volume` 等；tushare：沿用现「技术/基本面/统计」 |
| dependencies | 自研写依赖表名；tushare 写 `stk_factor_pro` |

`update-meta` 保持可扫自研 + tushare；冲突规则见 §4。

## 9. kline_stock_factor 定界

| 项 | 约定 |
|----|------|
| CLI | `uv run ./src/etl/cli.py stk-factor ...` |
| 用途 | Admin 看板完整性、少量 MACD/KDJ 展示 |
| 字段 | **冻结 14 列**，不再扩展到 93 |
| 与 Research | **不互相同步**；允许数值与 Parquet `macd` 等短期并存 |
| 研究代码 | 禁止 import / 查询该表做因子研究 |

在 [`docs/开发进度.md`](../../docs/开发进度.md) / 完整方案中已说明「与 Research 区别」；实现时在 `StkFactorStrategy` 模块 docstring 加一行定界即可。
