## 1. 目标

在 Apple M3 / AMD 9950X 等上把整机 CPU 吃满，缩短 `gtja191 compute` 墙钟时间。

**非目标：** 跨月并行、Ray/Dask、GPU。

---

## 2. 并行模型（锁定）

| 项 | 约定 |
|----|------|
| 维度 | **月内按 Alpha 分片**；月份仍串行 |
| 进程 | `ProcessPoolExecutor`（跨平台 spawn/fork 安全） |
| 面板 | 父进程建面板 → 写临时 Parquet → 各 worker 只读 |
| 写出 | 各 worker 写自己的 `factor/gtja_alpha{N}/dt=YYYYMM/`（目录独立，无锁） |
| Alpha143 | **父进程串行**（SELF）；不进池 |
| 失败 | 单 Alpha 失败只记日志，不拖垮整月 |

```text
logical = os.cpu_count() or 1
workers = clamp(logical - 1, 1, n_parallel_alphas)   # reserve=1
          可由 env GTJA191_WORKERS 或 CLI --workers 覆盖
POLARS_MAX_THREADS（每 worker）= max(1, logical // workers)
```

`workers=1` 时走原串行路径（便于对比/调试）。

## 3. 算子向量化（Phase B）

[`ops.py`](../../src/research/factor/gtja/ops.py) 热路径：

| 算子 | 策略 |
|------|------|
| `WMA` / `DECAYLINEAR` | 滞后加权 `sum_horizontal`（纯 Polars） |
| `TSRANK` / `HIGHDAY` / `LOWDAY` / `REGBETA` | 按 `ts_code` 的 NumPy `map_groups`（避免 `to_list`+纯 Python 滑窗） |
| 已向量化 | `DELAY`/`SUM`/`CORR`/`COVIANCE` 等保持 |

与多进程正交：降低每进程墙钟时间。

## 4. CLI / SSE / 环境变量

```bash
uv run ./src/research/cli.py gtja191 compute \
  [--start-month YYYYMM] [--end-month YYYYMM] \
  [--alpha N] [--force] [--workers N]
```

| 变量 / 参数 | 说明 |
|-------------|------|
| `GTJA191_WORKERS` | 覆盖自动探测的进程数 |
| `--workers` | CLI 覆盖（优先于 env） |
| SSE | `gtja191_compute` 可透传 `workers`；缺省自动探测 |

启动每月计算时日志打印：`workers=` / `polars_threads=`。

---

## 5. 临时文件

路径：`{WAREHOUSE_ROOT}/.tmp/gtja_panel_{YYYYMM}_{pid}.parquet`  
月份结束（成功或异常）后删除。

## 6. 自验

- `GTJA191_WORKERS=1` 与自动 workers 抽查同月同因子行数/抽样值一致  
- workers>1 时整机 CPU 显著高于串行
