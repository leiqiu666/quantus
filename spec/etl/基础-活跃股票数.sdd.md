# 基础 · 活跃股票数快照

## 目标

统一 Admin 量化数据源看板及各完整性校验的「活跃股票数」分母，消除 K 线类 / 市场类 / 财务类口径不一致。

## 两种活跃股票数

| 字段 | 含义 | 适用场景 |
|------|------|----------|
| `listed_count` | 当日未退市：`list_date <= date` 且 `(delist_date 为空 或 delist_date > date)`；排除 B 股 | 财报等按报告期/日历日统计的数据 |
| `trading_count` | 当日应交易：`listed_count` − 当日全天停牌股数（`suspend_type='S' AND suspend_timing=''`） | K 线、市场指标等交易日数据 |

停牌数 = `listed_count - trading_count`（仅开市日有意义）。

## 非开市日规则

若查询日不是 SSE 开市日（如报告期 `20060331` 落在周末），取 **≤ 该日最近一个开市日** 的 `trading_count`。

`listed_count` 始终按日历日本身计算，不做前移。

## 表结构

表名：`stock_active_count`

| 列 | 类型 | 说明 |
|----|------|------|
| `date_key` | VARCHAR(8) UNIQUE | YYYYMMDD：SSE 开市日或财报报告期 |
| `listed_count` | INT | 当日未退市 A 股数（不含 B 股） |
| `trading_count` | INT NULL | 当日应交易股数；非开市日为 NULL |

## 分层

```
CLI stock refresh-active-count
  → StockActiveCountStrategy.refresh_by_date_range
    → StockActiveCountWorkflow.build_rows / load_rows
      → Extract: stock_list, trade_cal, suspend_d
      → Transform: StockTransform.build_active_count_rows
      → Load: stock_active_count upsert
```

## CLI

```bash
uv run ./src/etl/cli.py stock refresh-active-count \
  --start-date 19900101 --end-date 20250705
```

默认起点：`settings.etl_start_date("stock_active_count")`（兜底 `ETL_DEFAULT_START_DATE`）。

## 前置依赖

- `stock_list` 已入库
- `stock_trade_calendar` 覆盖区间
- `stock_suspend` 覆盖区间内开市日（影响 `trading_count` 精度）

## 消费方

| 场景 | 分母字段 |
|------|----------|
| K 线 / 市场类 / 完整性 by-date | `trading_count`（非开市日按规则前移） |
| 财报报告期 / 完整性 by-period | `listed_count` |

读路径：`StockActiveCountService.resolve_trading_count` / `resolve_listed_count`。
