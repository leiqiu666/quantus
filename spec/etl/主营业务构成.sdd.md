# SDD · 主营业务构成

> **CLI 命令：** `financial_fina_mainbz pull-by-period`
> **交互菜单：** 【主营】主营业务构成全市场增量
> **源码入口：** `src/etl/cli.py`
> **Tushare 接口：** [`fina_mainbz`](https://tushare.pro/document/2?doc_id=81)

---

## 1. 概述

按报告期调用 Tushare `fina_mainbz_vip(period=, type=)` 全市场拉取主营业务构成（产品 P / 地区 D / 行业 I），upsert 到 PostgreSQL `financial_fina_mainbz` 表。

> 采用 **is_period** 完整性 + **报告期维度** VIP 拉取（同 forecast_vip）。需 **5000 积分**；每个报告期 × 每个 type 分页（100 行/页），约数十次请求/期，远少于逐股 5600+ 次。

### 触发方式

```bash
uv run ./src/etl/cli.py financial_fina_mainbz pull-by-period
uv run ./src/etl/cli.py financial_fina_mainbz pull-by-period --start-period 20100331
uv run ./src/etl/cli.py financial_fina_mainbz check-complete --start-period 20100331 --end-period 20241231
```

### 前置依赖

| 依赖 | 说明 |
|------|------|
| `TUSHARE_API_KEY` | 需 **5000 积分**（`fina_mainbz_vip`） |
| `FINA_MAINBZ_START_DATE` | floor（推荐 `20100331`） |
| `stock_list` | 全 A 股 ts_code |

---

## 3. 分层架构

```
CLI → FinaMainbzStrategy.pull_fina_mainbz_by_period
       ├─ CompletenessEngine.backfill_keys() → pending 报告期
       └─ for period in pending (tqdm):
            └─ FinaMainbzWorkflow.pull_fina_mainbz_period(period)
                 ├─ TushareFinaMainbzClient.fina_mainbz_vip (type=P/D/I, offset 分页)
                 └─ FinaMainbzLoad → financial_fina_mainbz
```
