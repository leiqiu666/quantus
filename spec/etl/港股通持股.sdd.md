# SDD · 港股通持股

> **CLI 命令：** `market_hk_hold pull-by-date-range`
> **源码入口：** `src/etl/cli.py`
> **Tushare 接口：** [`hk_hold`](https://tushare.pro/document/2?doc_id=188)

---

## 1. 概述

按 SSE 开市日逐日调用 Tushare `hk_hold`，拉取沪深港股通持股明细（沪股通/深股通/港股通），upsert 到 PostgreSQL `market_hk_hold` 表。完整性按全市场 ≥95% 股票数守门。

> 单次最多 3800 条，按 `trade_date` 拉取全市场。积分要求 2000+。限流 200/min。交易所自 2024-08-20 起停止发布日度北向资金，改为季度披露。

### 触发方式

```bash
uv run ./src/etl/cli.py market_hk_hold pull-by-date-range
uv run ./src/etl/cli.py market_hk_hold pull-by-date-range --start-date 20170317 --end-date 20240819
uv run ./src/etl/cli.py market_hk_hold check-complete
```

### 前置依赖

| 依赖 | 说明 |
|------|------|
| `TUSHARE_API_KEY` | 需 2000+ 积分 |
| `HK_HOLD_START_DATE` | floor（legacy env，推荐 `20170317` 深股通开通日） |
| `stock_trade_calendar`（SSE） | 开市日来源 |
| PostgreSQL | 目标库连接 |

## 3. 分层架构

```
CLI → HkHoldStrategy
       ├─ TradeCalStrategy.ensure_trade_cal(SSE)
       ├─ CompletenessEngine.backfill_keys()  ← 95% 全市场
       └─ for trade_date in pending:
            └─ HkHoldWorkflow.pull_hk_hold_by_date(trade_date)
                 ├─ HkHoldExtract → TushareHkHoldClient → ts.hk_hold(trade_date=)
                 └─ HkHoldLoad → bulk_upsert → market_hk_hold
```

**源码路径：** `src/etl/{strategy,workflow,extract,load,client}/market/market_hk_hold_*` + `src/entities/data_entities/market/market_hk_hold_entities.py`

---

## 4. 数据契约

| 字段 | 类型 | 说明 |
|------|------|------|
| code | str | 原始代码（空归一化为 `""`） |
| trade_date | str(8) | 交易日期 |
| ts_code | str | TS 代码 |
| name | str | 股票名称 |
| vol | int | 持股数量(股) |
| ratio | float | 持股占比(%) |
| exchange | str | SH/SZ/HK（空归一化为 `""`） |

**冲突键：** `trade_date, ts_code, exchange`

**完整性：** `source_name=market_hk_hold`，默认 95% 阈值

**Admin SSE：** `market_hk_hold_check` → `HkHoldStrategy.check_complete`

---

## 5. 边界

- `code` / `exchange` 的 None/NaN 在 `finalize_hk_hold` 归一化为 `""`，避免 upsert 冲突键失效
- 2024-08-20 后北向日度数据停更，历史区间仍可补拉
