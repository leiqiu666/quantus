# SDD · 申万行业分类

> **CLI 命令：** `index pull-classify-snapshot`
> **Tushare 接口：** [`index_classify`](https://tushare.pro/document/2?doc_id=181)

---

## 1. 概述

按 L1/L2/L3 × `src=SW2021` 调用 Tushare `index_classify`，upsert 到 `index_classify` 表。为申万行业因子、`index_member_all` 分批拉取提供 L1 代码列表。

拉取模式：**snapshot**（无完整性校验）。

## 2. 分层

```
CLI → IndexClassifyStrategy.pull_snapshot
  → IndexClassifyWorkflow.pull_index_classify_snapshot
    → for level in (L1,L2,L3): IndexClassifyExtract
    → IndexClassifyLoad → index_classify
```

## 3. 表结构

| 表名 | `index_classify` |
| 冲突键 | `(index_code, level, src)` |

字段：`index_code,industry_name,level,industry_code,src,is_pub,parent_code`

## 4. Admin / SSE

| 项 | 值 |
|----|-----|
| group_id | `index_month` |
| column_key | `index_classify` |
| sse_task_key | `index_classify_pull` |

## 5. 依赖

建议先拉 `index_classify`，再拉 `index_member_all`（后者按 L1 行业代码分批）。
