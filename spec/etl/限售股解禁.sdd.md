# SDD · 限售股解禁

> **CLI 命令：** `stock_share_float pull-by-date`；`stock_share_float check-complete`
> **Admin 大类：** 基础类 / 交易日（`group_id=stock_basic_trade_date`）
> **SSE task_key：** `stock_share_float_check`
> **源码入口：** `src/etl/cli.py`
> **Tushare 接口：** [`share_float`](https://tushare.pro/document/2?doc_id=160)

---

## 1. 概述

按 SSE 开市日作为解禁日候选，逐日调用 Tushare `share_float(float_date=)` 拉取全市场限售股解禁记录，upsert 到 PostgreSQL `stock_share_float` 表。为事件驱动因子（解禁冲击、流通盘变化）提供原始数据。

> 采用 **event_driven** 完整性（与 `financial_shareholder_top10` 同类）：某日 0 条也视为已拉取。限流 200/min。

### 触发方式

```bash
uv run ./src/etl/cli.py stock_share_float pull-by-date
uv run ./src/etl/cli.py stock_share_float pull-by-date --start-date 20100101 --end-date 20251231
uv run ./src/etl/cli.py stock_share_float check-complete
```

## 3. 分层架构

```
CLI → StockShareFloatStrategy.pull_share_float_by_date(start, end)
       ├─ TradeCalStrategy.ensure_trade_cal()
       ├─ CompletenessEngine.backfill_keys()          ← event_driven × float_date
       └─ for float_date in pending (tqdm):
            └─ StockShareFloatWorkflow.pull_share_float_by_float_date(float_date)
                 ├─ StockShareFloatExtract → TushareStockShareFloatClient
                 │    └─ ts.share_float(float_date=, offset/limit) → finalize
                 └─ StockShareFloatLoad → bulk_upsert_postgresql → stock_share_float
```

## 5. 业务规则

1. **按解禁日全市场：** 传 `float_date` 一次拉取当日全部解禁记录。
2. **NULL 归一化：** `holder_name` / `share_type` 的 None/NaN 在 finalize 时归一化为 `""`，保证 upsert 冲突键有效。
3. **完整性 event_driven：** `period_stock_count=1`，有数据或已 mark 均视为 resolved。
4. **空集 mark：** saved=0 时 `mark_date_pulled(float_date)`，避免反复空拉。
5. **Upsert 幂等：** 四列联合唯一。
