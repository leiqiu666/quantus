# SDD · 前十大股东

> **CLI 命令：** `shareholder pull-by-date`
> **交互菜单：** 【股东】前十大股东 by 公告日 区间增量 (shareholder pull-by-date)
> **源码入口：** `src/etl/cli.py`
> **Tushare 接口：** [`top10_holders`](https://tushare.pro/document/2?doc_id=61)

---

## 1. 概述

按 SSE 开市日作为公告日候选，逐日调用 Tushare `top10_holders(ann_date=)` 拉取全市场前十大股东数据（含持有数量和比例），upsert 到 PostgreSQL `financial_shareholder_top10` 表。为多因子模型提供前十大持股集中度、机构增减持、股东变动等筹码类因子。

> 与股东户数相同，采用 **event_driven** 完整性：某日 0 条也视为已拉取。积分要求 2000+。

### 触发方式

```bash
uv run ./src/etl/cli.py shareholder pull-by-date
uv run ./src/etl/cli.py shareholder pull-by-date --start-date 20100101 --end-date 20251231
uv run ./src/etl/cli.py
```

## 2. CLI 入口

| 项 | 值 |
|----|-----|
| Typer 子命令组 | `shareholder` |
| 命令名 | `pull-by-date` |
| 处理函数 | `shareholder_pull_by_date()` |
| 菜单 key | `shareholder-pull-by-date` |
| 菜单 label | `【股东】前十大股东 by 公告日 区间增量 (shareholder pull-by-date)` |

```python
@shareholder_strategy.command("pull-by-date")
def shareholder_pull_by_date(
    start_date: str | None = typer.Option(None, "--start-date"),
    end_date: str | None = typer.Option(None, "--end-date"),
) -> None:
    """按公告日逐日拉取 Tushare top10_holders 全市场并 upsert。"""
    total = ShareholderStrategy().pull_top10_holders_by_date(start_date=start_date, end_date=end_date)
    typer.echo(f"前十大股东累计写入 {total} 条")
```

## 3. 分层架构

```
CLI → ShareholderStrategy.pull_top10_holders_by_date(start, end)
       ├─ TradeCalStrategy.ensure_trade_cal()
       ├─ CompletenessEngine.backfill_keys()          ← event_driven × ann_date
       └─ for ann_date in pending (tqdm):
            └─ ShareholderWorkflow.pull_top10_by_ann_date(ann_date)
                 ├─ ShareholderExtract → TushareShareholderClient
                 │    └─ ts.top10_holders(ann_date=, offset/limit) → finalize
                 └─ ShareholderLoad → bulk_upsert_postgresql → financial_shareholder_top10
```

---

## 4. 逐步说明

| 步骤 | 位置 | 输入 | 处理 | 输出 |
|------|------|------|------|------|
| 1 | CLI | `--start-date` / `--end-date` | 实例化 Strategy | echo 总条数 |
| 2 | Strategy | floor / end | 缺省 → return 0 | — |
| 3 | Strategy | floor / end | `backfill_keys` 刷新快照并筛缺口 | pending ann_dates |
| 4 | Strategy | pending | tqdm 逐公告日调 Workflow | saved_count |
| 5 | Client | ann_date | ts.top10_holders(ann_date=) 分页 → finalize | DataFrame |
| 6 | Load | DataFrame | bulk_upsert_postgresql | upsert 条数 |
| 7 | Strategy | n == 0 | `mark_date_pulled(ann_date)` | 避免重复空拉 |

## 6. 业务规则

1. **按公告日全市场：** 传 `ann_date` 一次拉取当日全部公告的前十大股东，不再逐股 × 逐期双重循环。
2. **完整性 event_driven：** `period_stock_count=1`，有数据或已 mark 均视为 resolved。
3. **空集 mark：** saved=0 时 `mark_date_pulled`，避免反复空拉。
4. **Upsert 幂等：** `(ts_code, end_date, holder_name)` 联合唯一。
