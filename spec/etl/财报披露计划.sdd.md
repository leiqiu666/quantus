# SDD · 财报披露计划

> **CLI 命令：** `financial_disclosure_date pull-by-period`
> **交互菜单：** 【披露】财报披露计划全市场增量
> **源码入口：** `src/etl/cli.py`
> **Tushare 接口：** [`disclosure_date`](https://tushare.pro/document/2?doc_id=162)

---

## 1. 概述

按报告期调用 Tushare `disclosure_date(end_date=)` 拉取全市场财报披露计划（预计/实际披露日），upsert 到 PostgreSQL `financial_disclosure_date` 表。为 PEAD 等事件驱动策略提供披露时点。

> 采用 **is_period** 完整性（同 forecast）。积分要求 2000+。

### 触发方式

```bash
uv run ./src/etl/cli.py financial_disclosure_date pull-by-period
uv run ./src/etl/cli.py financial_disclosure_date pull-by-period --start-period 20100331
uv run ./src/etl/cli.py financial_disclosure_date check-complete --start-period 20100331 --end-period 20241231
```

### 前置依赖

| 依赖 | 说明 |
|------|------|
| `TUSHARE_API_KEY` | 需 2000+ 积分 |
| `DISCLOSURE_DATE_START_DATE` / `FINANCIAL_DISCLOSURE_DATE_START_DATE` | floor（推荐 `20100331`） |

---

## 3. 分层架构

```
CLI → DisclosureDateStrategy.pull_disclosure_date_by_period
       ├─ CompletenessEngine.backfill_keys()          ← is_period × end_date
       └─ for period in pending (tqdm):
            └─ DisclosureDateWorkflow.pull_disclosure_date_by_period(period)
                 ├─ DisclosureDateExtract → TushareDisclosureDateClient
                 │    └─ ts.disclosure_date(end_date=period)
                 └─ DisclosureDateLoad → financial_disclosure_date
```
