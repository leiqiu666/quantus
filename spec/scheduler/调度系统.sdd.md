# SDD · 调度系统

> **域：** 调度系统  
> **范围：** ETL CLI 交互菜单 25 条命令的定时编排与执行日志  
> **执行：** 默认随 `quantus-api` 在进程内启动 APScheduler；`quantus-scheduler` 为可选独立部署

---

## 1. 概述

Admin 配置「调度任务」：绑定若干 ETL 命令 + 执行周期；到点串行调 Strategy；PG 记录 run / step 日志。Admin「立即运行」走 API 后台线程，无需额外进程。

**启动方式（默认只开 API）：**

```bash
quantus-api   # SCHEDULER_ENABLED=true（默认），含定时 + 手动触发
curl http://localhost:8000/health   # scheduler: running
```

关闭内置调度：`SCHEDULER_ENABLED=false`。独立 Worker 仅在与 API 分离部署时使用，且 API 侧须关闭以免重复触发。

命令注册表单一事实源：[`src/scheduler/command_registry.py`](../../src/scheduler/command_registry.py)。

## 4. Worker 行为

1. 启动加载 `enabled=true` 的 job，注册 APScheduler CronTrigger。
2. 每 60s 对比 `max(updated_at)`，变化则 reload。
3. 触发前：`run_on_trading_day` 且非开市日 → skipped run。
4. 同 job 有 running run → skip。
5. 串行执行命令；单步失败继续后续；run 终态 success / partial / failed。

**禁止** subprocess 调 `cli.py`；走 [`src/scheduler/runner.py`](../../src/scheduler/runner.py) → registry runner → Strategy。

---

## 5. 与 etl_sse_registry 关系

- SSE：`task_key` + 日期区间，Admin 看板补位。
- 调度：`command_key`，无参增量 pull（菜单路径）。
- key 空间分离，底层共享 Strategy。
