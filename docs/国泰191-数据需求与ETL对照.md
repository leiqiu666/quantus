## 0. 小白速读：这库到底要什么？

国泰君安 191（GTJA 191 / Alpha191）是一套**纯技术面价量因子**公式库。  
绝大多数公式只用到个股日线的：

| 公式符号 | 含义 | 怎么来 |
|----------|------|--------|
| `OPEN/HIGH/LOW/CLOSE` | 开高低收 | 日线行情 |
| `VOLUME` / `VOL` | 成交量 | 日线行情 |
| `AMOUNT` | 成交额 | 日线行情 |
| `VWAP` | 成交量加权均价 | 通常用 `成交额/成交量` **推导**，不是独立行情字段 |
| `RET` | 日收益率 | `CLOSE/昨收 - 1` **推导** |
| `DTM/DBM/LD/HD/TR` | 涨跌动能/真实波幅类中间量 | 由 OHLC **推导** |
| `BANCHMARKINDEX*` | 基准指数开收盘 | 指数日线 |
| `MKT/SMB/HML` | Fama-French 三因子 | **无现成 Tushare 接口**，需自建 |
| `SELF` | 因子自身递推 | 计算逻辑，不需额外数据 |

**结论一句话**：算 191 的主菜是「个股日 K +（可选）复权 + 基准指数日线」；财务/资金流/龙虎榜等**不是**这套公式的输入。

## 2. 需要特别说明的两项

### 2.1 VWAP：没有单独 ETL，但原料已齐

Tushare `daily` **不返回** `vwap` 字段。业界常见做法：

```text
VWAP ≈ amount / vol
# 注意单位：amount 为千元，vol 为手（1 手 = 100 股）
# 换算成「元/股」时需统一量纲，例如：
#   VWAP = (amount * 1000) / (vol * 100) = amount / vol * 10
```

可选但**不必**为 191 单独接的接口：

| API | 字段 | ETL | 文档 |
|-----|------|-----|------|
| `bak_daily` | `avg_price`（平均价） | ❌ 未接入 | https://tushare.pro/document/2?doc_id=255 |

**建议**：计算层用 `kline_daily.amount/vol` 推导即可，不必为 VWAP 新开 ETL。

### 2.2 Alpha30 的 MKT / SMB / HML：唯一真正缺口

```text
Alpha30: WMA((REGRESI(CLOSE/DELAY(CLOSE)-1, MKT, SMB, HML, 60))^2, 20)
```

这是把个股收益对 **Fama-French 三因子**做回归残差再平滑。  
Tushare **没有**现成 `mkt/smb/hml` 接口，需要自建：

| 因子 | 含义 | 自建所需原料 | 原料 ETL |
|------|------|--------------|----------|
| `MKT` | 市场超额收益 | 基准指数收益（如沪深300/上证）− 无风险利率（可简化为 0） | ✅ `index_daily` |
| `SMB` | 小盘−大盘 | 按市值分组的多空组合收益 | ✅ `market_daily_basic`（`total_mv`/`circ_mv`）+ 日收益 |
| `HML` | 高账面市值比−低 | 按 BM 分组的多空组合收益 | ⚠️ 需净资产：财报已接，但**无现成 BM/FF 管线** |

→ **191 全库里只有 Alpha30 卡在「因子成品未建」**；其余 190 个在数据层可算。

## 4. 191 **不需要**、但项目里已有的数据（避免误解）

下列 ETL 对量化很有用，但**不是**国泰 191 公式的输入：

| 已接入数据源 | Tushare API | 为何 191 不用 |
|--------------|-------------|---------------|
| 财报三表 / 财务指标 | `income` 等 | 191 无基本面字段 |
| 每日估值 PE/PB 等 | `daily_basic` | 公式未用（仅 Alpha30 自建 HML 时可能间接用市值/BM） |
| 资金流向 / 两融 / 北向 | `moneyflow` 等 | 价量库不涉及 |
| 龙虎榜 / 大宗 | `top_list` 等 | 同上 |
| 股东户数 / 十大股东 | `stk_holdernumber` 等 | 同上 |
| 技术因子成品 `stk_factor_pro` | `stk_factor_pro` | 191 是另一套公式，不消费该表 |

## 5. 总结

1. **国泰 191 = 价量技术因子库**，核心只要个股日线 OHLCV + 成交额；VWAP/RET/DTM 等均可推导。  
2. **当前 ETL 对 191 的覆盖极高**：`daily`（含 amount）+ `adj_factor` + `index_daily` + 日历/列表/停复牌均已接入。  
3. **唯一明确缺口**：Alpha30 所需的 **MKT/SMB/HML 成品序列**（Tushare 无 API，需 Research 层自建）。  
4. **VWAP 不是缺口**：没有独立字段/表，用已有 `amount/vol` 在计算层生成即可。  
5. **落地建议**：先用 `kline_daily`（建议复权）实现 190 个因子；Alpha30 单独排期做 FF 三因子管线；不必为 191 再接财务/资金流类 API。
