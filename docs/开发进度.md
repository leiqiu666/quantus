### Typer 子命令（不在交互菜单）

| CLI 命令 | 用途 |
|----------|------|
| `report check-report-complete` | 财报四表宏观 → 逐股缺期补拉 → 宏观 |
| `kline check-complete` | K 线三维度宏观 → 逐股缺日补拉 → 宏观 |
| `<group> check-complete` | 各 P0~P3 数据源微观补拉（daily-basic / market_dividend / stk-factor / …） |
| `kline check-daily-complete` / `check-adj-factor-complete` / `check-stk-limit-complete` | `check-complete` 的单维度变体 |
| `kline update-adj-factor-period-count` / `update-stk-limit-period-count` | 与 `update-daily-period-count` 同实现 |
| `kline pull-daily-by-date` / `pull-adj-factor-by-date` / `pull-stk-limit-by-date` | 单日拉取，date-range 内部使用 |

## 6. 关键环境变量

来源：[`src/common/setting.py`](../src/common/setting.py) + `.env`

**ETL 增量起点规则：**

| 优先级 | 配置 | 说明 |
|--------|------|------|
| 1 | `{DB表名}_START_DATE` | 映射后 PG 表名大写，如 `MARKET_DAILY_BASIC_START_DATE` |
| 2 | legacy 旧名 | 如 `DAILY_BASIC_START_DATE`（兼容，见 `src/common/etl_start.py`） |
| 3 | 特殊默认 | 仅 `kline_stk_limit` → `20090601` |
| 4 | `ETL_DEFAULT_START_DATE` | 全局兜底，默认 `19901219` |

按月增量用 `{DB表名}_START_MONTH`（如 `INDEX_WEIGHT_START_MONTH`）。代码侧统一 `settings.etl_start_date("表名")` / `settings.etl_start_month("index_weight")`。

| 变量 | 默认 | 用途 |
|------|------|------|
| `ETL_DEFAULT_START_DATE` | `19901219` | 未单独配置的 ETL 起始日 |
| `POSTGRESQL_*` | — | PostgreSQL 连接 |
| `POSTGRESQL_POOL_SIZE` / `POSTGRESQL_MAX_OVERFLOW` | `10` / `20` | SQLAlchemy 连接池（`Database` 单例 engine） |
| `VITE_ETL_SSE_MAX_CONCURRENT` | `5` | Admin SSE 补位最大并行数（根目录 `.env`，需重启 `pnpm dev`） |
| `TUSHARE_API_KEY` | — | Tushare Pro 鉴权 |
| `WAREHOUSE_ROOT` | `./data/warehouse` | Parquet 仓库根 |
| `KLINE_*_SOURCES` / `REPORT_*_SOURCES` | — | 各维度数据源链 |
| `TDX_QUANT_ENABLED` / `TDX_*` | — | 通达信本地客户端 + HTTP 代理 |

各表可选起点完整清单见 [`docs/ETL模块分类与命名规范.md`](ETL模块分类与命名规范.md) 映射表 + [`src/common/etl_start.py`](../src/common/etl_start.py)。
